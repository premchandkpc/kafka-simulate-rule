# VM Specification

## 1. Purpose

The FlowRule Virtual Machine executes compiled bytecode modules deterministically. It is an embeddable, stateless interpreter designed for low-latency message routing. The VM has no knowledge of transports, plugins, or storage — it only executes instructions and delegates external calls via interfaces.

## 2. Architecture

## 3. Execution Model

### 3.1 Dispatch Loop
```
function Execute(ctx, msg, module):
    ip = 0
    while ip < len(module.instructions):
        instr = module.instructions[ip]
        switch instr.opcode:
            NOP:       break
            NEXT:      callTarget(ctx, msg, module, instr)
            PARALLEL:  fanOut(ctx, msg, module, instr)
            COLLECT:   syncParallel(msg)
            FALLBACK:  tryFallback(ctx, msg, module, instr)
            GATE:      evalGate(msg, module, instr)
            PIPE:      ip = skipToEnd(instructions, ip)
            EMIT:      fireAndForget(ctx, msg, module, instr)
            DROP:      return nil (message dropped)
            MAP:       transform(msg, module, instr)
            KEY:       extractKey(msg, module, instr)
            SPLIT:     extractKey(msg, module, instr)  // same as KEY for v1
            BUFFER:    return error (engine-level)
            JUMP:      ip = instr.arg1; continue
            JUMP_IF:   if msg.failed: ip = instr.arg1; continue
            JUMP_IFN:  if not msg.failed: ip = instr.arg1; continue
        ip++
    return nil
```

### 3.2 Instruction Pointer
- `ip` is a `uint32` index into the instruction array
- Default advancement: `ip++`
- Jump instructions override: `ip = instr.arg1`
- Skip operations (GATE false, PIPE): set `ip` to next PIPE or end

### 3.3 Execution Context
Each execution has:
- **Message**: mutable state flowing through instructions
  - `Body []byte` — original message payload
  - `LastResponse []byte` — result of last NEXT/PARALLEL
  - `Headers map[string]string` — metadata
  - `Failed bool` — failure flag
  - `Errors []StageError` — accumulated errors
  - `HopCount int` — number of successful deliveries
  - `PartitionKey string` — extracted by KEY/SPLIT
  - `Deadline time.Time` — message timeout
- **Module**: the loaded bytecode (immutable)
- **Interfaces**: Caller, Emitter, Breaker, Credit

## 4. Opcode Behaviors

### 4.1 NEXT
1. Resolve URL from constant pool via instruction args
2. Check credit: `Credit.CanSend(target)` — block/error if exhausted
3. Check breaker: `Breaker.Allow(target)` — fail-fast if open
4. Apply timeout from instruction flags if present
5. Execute with retry loop:
   ```
   for attempt = 0; attempt <= retryN; attempt++:
       resp, err = Caller.Call(ctx, target, body)
       if err == nil:
           msg.LastResponse = resp
           msg.HopCount++
           msg.Failed = false
           break
       Breaker.RecordFailure(target)
       if attempt < retryN: backoff and wait
   ```
6. On final failure: set `msg.Failed`, record error, check fallback

### 4.2 PARALLEL
1. Load target list from module
2. Launch goroutine per target
3. Collect all results
4. On any failure: set `msg.Failed`
5. Default 30s timeout per branch

### 4.3 COLLECT
1. If `msg.Failed`: check for fallback
2. Otherwise: increment `HopCount` (marks parallel sync point)

### 4.4 FALLBACK
1. If `msg.Failed`: call target, clear failure state on success
2. Otherwise: no-op (skip)

### 4.5 GATE
1. Extract field value from message (Body or LastResponse)
2. Compare with operator and value
3. If false: `ip = skipToPipe`
4. If true: continue to next instruction

### 4.6 PIPE
1. `ip = skipToEnd` (skip to matching pipe or end)

### 4.7 EMIT
1. Load target list
2. For each target: `go Emitter.Emit(ctx, target, body)`
3. Context timeout: 5s
4. Fire-and-forget: no error propagation

### 4.8 MAP
1. Load map expression from module
2. Evaluate against body (LastResponse or Body)
3. Set `msg.LastResponse` to result

### 4.9 KEY/SPLIT
1. Extract field value from LastResponse
2. Set `msg.PartitionKey`

## 5. Gate Evaluation

```
function evalGate(msg, field, operator, value):
    body = msg.LastResponse ?? msg.Body
    fieldValue = extractJSONField(body, field)
    return compare(fieldValue, operator, value)
```

### 5.1 JSON Field Extraction
- Parse body as JSON
- Navigate dot-separated path
- Coerce value to string for comparison
- Numbers: float64 → string (integer if whole number)
- Booleans: "true" or "false"
- Objects/arrays: JSON string

### 5.2 Comparison
- `==` / `!=`: string equality
- `>` / `<` / `>=` / `<=`: parse both as float64
- `contains`: substring match

## 6. Fault Isolation

- All external calls have context deadlines
- Circuit breaker prevents calls to failing targets
- Credit system prevents overwhelming targets
- Timeout on every NEXT (default 30s, per-rule configurable)
- Backpressure: credit exhaustion blocks the worker, not the process
- Plugin execution timeout: 5s
- VM never panics: recoverable errors set Failed flag
- Unrecoverable errors (invalid bytecode, bounds) return error immediately

## 7. Determinism Guarantees

- Same module + same message = same execution trace
- No random values in VM (jitter is in transport layer, outside VM)
- No global state
- No runtime reflection
- No code generation
